# 团队拉取与启动验收

> 状态：现行验收步骤
> 最后核验：2026-07-22
> 维护人：A 维护公共启动方式；各成员补充自己模块的特殊前提
> 前提：团队只使用本 GitHub 仓库；后端位于 `backend/`，前端位于 `frontend/`，两者是并列的独立工程

## 1. 拉取与数据库初始化

```bash
git clone https://github.com/YuCPbit/green-channel-backend.git
cd green-channel-backend
mysql -u root -p < docs/03-数据库/数据库设计.sql
mysql -u root -p < docs/03-数据库/数据库初始化数据.sql
cd backend
mvn test
```

数据库密码只通过 `DB_PASSWORD` 环境变量提供。所有后端服务还必须使用相同且至少 32 字符的
`APP_AUTH_TOKEN_SECRET`，否则 platform-service 签发的令牌不能被其他服务验证。

## 2. 启动六个服务

分别在六个终端执行：

```bash
# 当前目录：backend/
DB_PASSWORD='本机密码' APP_AUTH_TOKEN_SECRET='至少32字符且各服务相同' mvn -pl platform-service -am spring-boot:run
DB_PASSWORD='本机密码' APP_AUTH_TOKEN_SECRET='至少32字符且各服务相同' mvn -pl gift-service -am spring-boot:run
DB_PASSWORD='本机密码' APP_AUTH_TOKEN_SECRET='至少32字符且各服务相同' mvn -pl subsidy-service -am spring-boot:run
DB_PASSWORD='本机密码' APP_AUTH_TOKEN_SECRET='至少32字符且各服务相同' mvn -pl workstudy-service -am spring-boot:run
DB_PASSWORD='本机密码' APP_AUTH_TOKEN_SECRET='至少32字符且各服务相同' mvn -pl dashboard-service -am spring-boot:run
mvn -pl gateway-service spring-boot:run
```

前端和测试工具只访问网关 `http://localhost:8080`。不要为了联调修改 8081—8085 的固定端口分配。

## 3. 启动前端

```bash
cd ../frontend
npm ci
npm run dev
```

浏览器访问 `http://localhost:5173`。开发代理将 `/api` 转发到网关 8080；网关默认允许该来源。
部署到其他域名时，在启动网关前设置 `APP_CORS_ALLOWED_ORIGINS`，多个来源使用英文逗号分隔。

## 4. 自动验收

```bash
cd ../backend
SMOKE_USERNAME='开发账号' SMOKE_PASSWORD='开发密码' ./scripts/smoke-api.sh
```

脚本验证平台健康、登录、platform 验签、dashboard 路由和跨服务验签。随后按各成员模块执行真实业务流程。

## 5. 成员验收回报

每位成员在 Jira 对应任务中记录：

- 操作系统与 Java、Maven、MySQL 版本；
- `git rev-parse --short HEAD` 的结果；
- `cd backend && mvn test` 和冒烟脚本结果；
- `cd frontend && npm ci && npm run build` 的结果；
- 自己模块的正常流程、异常流程和权限流程；
- 已脱敏的失败日志；
- 对其他模块造成的接口或数据库变更。
