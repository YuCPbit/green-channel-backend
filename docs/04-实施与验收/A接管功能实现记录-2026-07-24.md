## 3. A 新增的功能闭环

### 3.1 资助方案（Jira 15）

- 学校端创建、修改草稿，配置资金来源、固定/区间金额、名额、期限和准入条件。
- 草稿可试运行估算、发布，已发布方案可下线。
- 学生只会查到当前有效且已发布的方案。

### 3.2 学生申诉和管理员处理（Jira 16、17）

- 仅允许当前学生针对本人出现“不通过”审核记录的绿色通道或补助申请申诉。
- 申诉窗口读取 `APPEAL_WINDOW_DAYS`，默认 3 天。
- 附件只能绑定当前用户已上传的附件，防止跨用户引用。
- 申诉自动回到作出“不通过”的辅导员/学院/学校节点。
- 支持待受理、处理中、申诉成立、驳回、退回补充，并记录处理人和结论。

### 3.3 调岗与离岗（Jira 45）

- 学生从本人当前在岗记录发起调岗或离岗申请，同一在岗记录禁止重复待办。
- 调岗必须选择不同且仍有名额的上架岗位。
- 审批通过后以事务方式释放旧岗位名额、占用新岗位名额、保存旧在岗历史并生成新在岗关系和协议。
- 离岗审批通过后才改变在岗状态并释放名额，替代原来无需审批的直接变更。

### 3.4 满意度问卷（Jira 54）

- 学校创建并发布指定业务/批次的 5 分制问卷。
- 学生在开放日期内填写一次评分和建议，数据库唯一键防重复提交。
- 管理端查看答卷数、平均分、1—5 分分布、近期建议和简单高频词。

### 3.5 勤工助学主流程联调（D 主流程，A 接管整合）

- 没有整条合入 D 的 `dev/workstudy` 前端分支，因为它会删除当前绿色通道、补助、申诉和资金台账页面。
- 在独立前端仓库选择性接入批次、岗位、报名、面试、录用、考勤、补卡、薪酬、协议页面和 API。
- 后端补充录用、考勤、薪酬、协议列表接口，使页面不再依赖手填业务 ID。
- 修复学生报名读取 `username` 并信任请求体 `studentId` 的越权问题，统一使用登录态 `CurrentUser.id`。
- 学生查询岗位、申请详情、评价等接口增加本人/公开状态范围限制。
- 补齐五类演示角色的勤工助学菜单和接口权限，已有数据库执行勤工助学增量迁移即可生效。

### 3.6 礼包核销与协议修复

- 礼包正常核销、异常登记和补发改为携带旧状态条件的原子更新，阻止并发重复核销。
- 未知领取码统一返回 `40400`，重复/状态冲突统一返回 `40900`；异常登记强制填写原因。
- 删除已失效的旧 `StudentApplyService.pickup` 测试，按当前 `GiftReviewService` 接口补 5 个核销测试。
- 协议控制器不再强转实现类；协议编号正文与记录保持一致，续签会延长到期日并使用并发条件更新。
- 学生只能查看和签署本人协议，管理端新增协议列表和可选新到期日续签。

### 3.7 登录身份、权限与数据范围加固

- 礼包申请不再接收前端传入的学生身份、申请编号和审核状态；服务端按登录用户映射
  `gc_student.id`，并校验只能查看、修改本人申请。
- 礼包批次、物品、名额、核销、补录和各级审核接口补齐菜单/按钮权限，核销人与补录人
  统一从登录态读取，客户端不能伪造操作人。
- 勤工报名、录用、考勤、薪酬、协议和评价统一区分“用户 ID”与“学生档案 ID”；
  评价的学生身份由录用记录反查，不再信任请求体。
- 学院/用工部门只能提交、修改、下架本人发布的岗位，只能查看和确认本人岗位的考勤；
  学校与系统管理员保留全局管理能力。
- 评价低分预警不再写死管理员 ID 或仅打印 Mock 日志，改为向所有启用的学校资助中心账号
  写入真实站内消息。

### 3.8 系统管理、新生、消息和报表页面

- 前端补齐“学生管理 / 新生管理”，学校端可查询、新增、编辑、删除、下载模板和导入 Excel；
  辅导员端复用所带学生检索接口。
- 补齐“消息中心”，支持未读筛选、单条已读和全部已读。
- 补齐“用户管理”，支持账号创建、启停、基本信息与角色分配、密码重置。
- 补齐“角色权限”，支持创建/编辑角色并分配菜单级、按钮级权限。
- 补齐“字典参数”，支持字典项增删改，并可维护申诉窗口、勤工最低时薪和每周工时上限等
  系统参数。
- 补齐“事务类型配置”，学校或系统管理员可维护辅导员事务类型及审批级数。
- 补齐“接口监控 / 操作日志”列表和“数据看板 / 学院报表”；学校看板支持学生资助、
  勤工助学标准报表预览和 Excel 导出。
