# Jira 与 Git 同步规则

每完成一项可验收工作，必须在同一次工作收尾中同时更新 GitHub 和 Jira。

## GitHub 更新

1. 每个提交只对应一个 Jira Story、Task 或 Sub-task。
2. 提交标题包含 Jira 编号或业务编号，例如：`feat(GCS-S01): add RBAC permission checks`。
3. 推送前至少运行与改动相关的测试或构建。
4. 禁止提交密码、Token、邮箱验证码、数据库真实凭据、个人联系方式和未脱敏日志。

## Jira 更新

1. 开始编码时把任务从“待办”改为“处理中”。
2. 工作较大时拆为可独立验收的子任务。
3. 完成后添加评论，至少包含：
   - 完成内容；
   - Git 提交号；
   - 测试或构建结果；
   - 未完成项或风险；
   - 对其他成员的接口影响。
4. 只有功能和验收均通过后才能改为“完成”。
5. 接口发生变更时，先更新公共接口约定，再通知依赖成员。

## 报表数据要求

- Story Point 填写在 Story 上，子任务不重复计点。
- Sprint 开始后不要把未完成工作直接标记为完成。
- 每天至少更新一次状态，使燃耗图、累积流量图和 Sprint 报告保持可信。
- 需求新增或范围变化必须通过新任务或子任务体现，不能只在群聊中口头变更。
