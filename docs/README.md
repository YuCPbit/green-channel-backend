## 维护规则

1. 新的长期有效规则只能放入 `01-现行规范`。
2. 需求和设计草案放入 `02-需求与设计`，必须写清“已实现/部分实现/未实现”。
3. 数据库可执行文件只放入 `03-数据库`。
4. 一次性审计、迁移记录和验收报告放入 `04-实施与验收`，文件名带日期。
5. 不再有效的文档移动到 `99-历史归档`，并在开头写明归档日期和失效原因。
6. 每份现行 Markdown 文档必须在标题下写状态、最后核验日期和维护人。
