# 服务接口与调用示例

> 状态：现行调用入口
> 最后核验：2026-07-22（对应 main 与 PR #8）
> 维护人：A 维护公共部分，各成员维护自己模块的接口示例

## 统一入口与端口

前端只访问网关 `http://localhost:8080`，不要直接记各业务服务端口。

| 路径 | 目标服务 | 默认地址 |
| --- | --- | --- |
| `/api/gift/**` | gift-service | `http://localhost:8082` |
| `/api/subsidy/**` | subsidy-service | `http://localhost:8083` |
| `/api/workstudy/**` | workstudy-service | `http://localhost:8084` |
| `/api/dashboard/**`、`/api/report/**`、`/ws/dashboard/**` | dashboard-service | `http://localhost:8085` |
| 其余 `/api/**` | platform-service | `http://localhost:8081` |

部署时分别用 `PLATFORM_SERVICE_URL`、`GIFT_SERVICE_URL`、`SUBSIDY_SERVICE_URL`、
`WORKSTUDY_SERVICE_URL` 和 `DASHBOARD_SERVICE_URL` 覆盖目标地址。

## 公共契约

所有 JSON 接口统一返回：

```json
{
  "code": 0,
  "message": "success",
  "data": {},
  "timestamp": "2026-07-22T03:00:00Z"
}
```

常用错误码：`40000` 请求不合法、`40100` 未登录或令牌失效、`40300` 无权限、
`40400` 资源不存在、`40900` 状态冲突、`50000` 系统异常、`50300` 外部服务不可用。

登录由 platform-service 签发 8 小时有效的 HMAC 令牌。各服务通过 `common-api` 验签并恢复
`CurrentUser`；所有服务必须使用相同且至少 32 字符的 `APP_AUTH_TOKEN_SECRET`。业务接口不得
接受客户端传入的 `userId`、`studentId` 或角色请求头作为身份依据。

## 可直接执行的 curl 示例

设置网关地址并检查平台服务：

```bash
BASE_URL=http://localhost:8080
curl -fsS "$BASE_URL/api/health"
```

登录并复制响应 `data.token`：

```bash
curl -fsS -X POST "$BASE_URL/api/auth/login" \
  -H 'Content-Type: application/json' \
  -d '{"username":"你的账号","password":"你的密码"}'

TOKEN='把上一步 data.token 填到这里'
curl -fsS "$BASE_URL/api/auth/me" -H "Authorization: Bearer $TOKEN"
```

通过网关调用各业务模块：

```bash
curl -fsS "$BASE_URL/api/gift/batch/page?current=1&size=10" \
  -H "Authorization: Bearer $TOKEN"

curl -fsS "$BASE_URL/api/subsidy/batches/available" \
  -H "Authorization: Bearer $TOKEN"

curl -fsS "$BASE_URL/api/workstudy/position/list" \
  -H "Authorization: Bearer $TOKEN"

curl -fsS "$BASE_URL/api/dashboard/stats?module=workstudy" \
  -H "Authorization: Bearer $TOKEN"

curl -fsS "$BASE_URL/api/dashboard/chart?module=workstudy&type=position-stats" \
  -H "Authorization: Bearer $TOKEN"
```

预览勤工助学岗位统计报表。`batchId` 会被解析为整数并通过 MyBatis 参数绑定，不能传 SQL 片段：

```bash
curl -fsS -X POST "$BASE_URL/api/report/preview" \
  -H "Authorization: Bearer $TOKEN" \
  -H 'Content-Type: application/json' \
  -d '{
    "module":"workstudy",
    "reportType":"position-stat",
    "templateName":"勤工助学岗位统计",
    "fields":["dept_name","position_count","apply_count","on_job_rate"],
    "filters":{"batchId":1}
  }'
```

导出同一报表：

```bash
curl -fsS -X POST "$BASE_URL/api/report/export" \
  -H "Authorization: Bearer $TOKEN" \
  -H 'Content-Type: application/json' \
  -d '{"module":"workstudy","reportType":"position-stat","templateName":"岗位统计"}' \
  -o 岗位统计.xlsx
```

## 自动冒烟检查

数据库和六个可运行服务均启动后执行：

```bash
SMOKE_USERNAME='你的账号' SMOKE_PASSWORD='你的密码' ./scripts/smoke-api.sh
```

脚本依次验证网关健康检查、登录、跨服务令牌解析和 dashboard 路由。任一步返回非 2xx、
业务 `code` 非 0 或令牌缺失都会立即失败。
